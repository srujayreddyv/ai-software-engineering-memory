# Review

## Summary

Summarize what changed and why.

## Correctness

Assess whether the implementation satisfies the task and handles relevant edge cases.

## Security

Assess authentication, authorization, data exposure, input validation, dependency risk, and sensitive data handling.

## Performance

Assess latency, throughput, resource usage, query behavior, bundle size, and scalability impact where relevant.

## Maintainability

Assess clarity, consistency with existing patterns, architectural boundaries, coupling, and unnecessary complexity.

## Backward Compatibility

Assess API compatibility, data compatibility, migration safety, user-facing behavior changes, and integration impact.

## Test Coverage

List tests added, updated, or run. Note meaningful gaps.

## Known Risks

Document remaining risks, assumptions, or follow-up work.

## Recommendations

List concrete actions required or suggested before merge, release, or handoff.

## Final Status

Allowed values:

* Approved
* Approved With Concerns
* Needs Changes
